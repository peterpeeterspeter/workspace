## How Provably Fair Works

### The Concept

Provably fair is a cryptographic system that proves the casino couldn't have rigged the crash outcome after you placed your bet.

**How it works:**
1. **Before you bet:** Casino generates a random server seed and shares a HASH (encrypted version) with you
2. **You place your bet:** You provide or approve a client seed (your random input)
3. **Crash happens:** Server seed + client seed + nonce (round number) = crash point
4. **After the round:** Casino reveals the actual server seed (you can verify it matches the hash from step 1)
5. **You verify:** You can recreate the crash point yourself using the seeds ‚Äî if it matches, game is fair

**Why it's trustworthy:**
- Casino commits to outcome BEFORE you bet (hash commitment)
- They can't change server seed after you bet (hash won't match)
- You contribute randomness (client seed)
- You can independently verify the math

### What to Look For (Transparent vs Opaque)

**Transparent Casinos (Good Signs):**
- ‚úÖ Server seed hash is displayed BEFORE you bet
- ‚úÖ Client seed is generated by YOU (not casino)
- ‚úÖ Verification tool is built-in (one-click verify)
- ‚úÖ You can run 1,000+ simulations to verify RTP
- ‚úÖ Support can explain provably fair in detail

**Opaque Casinos (Red Flags):**
- ‚ùå No hash disclosure (they can change outcome after you bet)
- ‚ùå Client seed is generated by casino (they know result in advance)
- ‚ùå No verification tool (you have to "trust" them)
- ‚ùå Support can't explain provably fair (they don't understand it)
- ‚ùå Vague "we're fair" claims without proof

## Fake Provably Fair: How Casinos Cheat

### Scam #1: Hash is Fake

**How it works:**
- Casino displays a hash
- But that hash isn't actually used in the game
- Real game uses a different (rigged) seed

**Detection:**
- Verify crash point ‚Äî if calculated result ‚â† actual result, hash is fake
- Run 1,000 rounds ‚Äî if RTP is 85-90%, provably fair is fake

**Real example (2024):**
A casino showed server seed hashes but didn't actually use them in game logic. Player verification revealed calculated crash points never matched actual results. The casino was faking transparency while rigging outcomes.

### Scam #2: Client Seed is Predictable

**How it works:**
- Casino generates client seed for you
- Seed looks random but follows a pattern
- Casino knows your seed in advance ‚Üí knows crash point

**Detection:**
- If client seed is generated by casino (not you), it's a red flag
- Test: Create your own random seed ‚Äî if casino won't accept it, they're cheating

### Scam #3: Verification Tool is Rigged

**How it works:**
- Verification tool ALWAYS shows "verified"
- Even if you input garbage data, it says "verified"
- Tool is a prop, not actual verification

**Detection:**
- Test verification tool with fake data
- If it still says "verified" regardless of input, tool is rigged
- Example: Input "AAAA" as server seed ‚Äî should produce error, not "verified"

## FAQ

### Q: How do I verify crash game fairness?

**A:** 5-step process:
1. **Find a bet** in your bet history
2. **Copy seed data** (server seed, client seed, nonce)
3. **Use verification tool** (built-in or script)
4. **Calculate crash point** from seed data
5. **Compare to actual crash point** ‚Äî if matches, game is fair

### Q: Can crash games be rigged even if they're "provably fair"?

**A:** YES. Scammers fake provably fair in 3 ways:
1. **Fake hash** (display hash but don't use it in game)

**How to detect:** Run 1,000 rounds, calculate actual RTP. If it's 85-90%, provably fair is fake. Legit crash games pay 96-97% RTP.

**If casino ghosts you or bans you:** Blacklist them and warn others. Failed verification = rigged game.
