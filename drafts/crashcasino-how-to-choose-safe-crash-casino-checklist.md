## Provably Fair Verification

Provably fair systems are essential for crash games. Here's how to verify a casino's crash game is truly fair.

### What is Provably Fair?

Provably fair is a cryptographic system that allows players to verify game outcomes weren't manipulated. In crash games, each round's crash point is determined before betting closes, encrypted, and revealed after the round ends.

### How Provably Fair Works (Simplified)

1. **Before round:** Casino generates random seed and encrypts it (the "hash")
2. **Betting:** Players place bets without knowing the crash point
3. **After round:** Casino reveals the original seed
4. **Verification:** Players use the seed to calculate the crash point and verify it matches

This prevents the casino from changing the outcome after seeing player bets.

### Verifying Provably Fair: Step-by-Step

**Step 1: Find the Verification Tool**
- Look for "Provably Fair" or "Fairness" link in game footer
- Some casinos have dedicated verification pages

**Step 2: Get Round Data**
- Note the round ID or number
- Copy the server seed and client seed for that round

**Step 3: Use Verification Tool**
- Input the seeds into the casino's verification calculator
- Tool will calculate the crash point
- Verify it matches the actual crash point from the round

**Step 4: Manual Verification (Optional)**
- Some casinos provide code/scripts for independent verification
- Requires programming knowledge

### What to Look For

‚úÖ **Legitimate provably fair systems:**
- Hash is published BEFORE the round starts
- Seeds can be exported for verification
- Verification process is clearly explained
- Third-party auditors have verified the system

‚ùå **Fake provably fair systems:**
- Hash only appears AFTER round ends (allows manipulation)
- Verification tool is missing or broken
- Casino refuses to provide seeds for verification
