## How Provably Fair Works (The Concept)

**The basic idea:** The casino commits to the crash point *before* you place your bet. You can verify after the round that the outcome wasn't manipulated.

**How it works (simplified):**
1. **Server seed** (casino's secret) + **Client seed** (your input) + **Nonce** (round number) = **Crash point**
2. The casino hashes the server seed *before* the round (so they can't change it later)
3. After the round, the casino reveals the server seed
4. You can plug server seed + client seed + nonce into a verification tool
5. The tool calculates the crash point ‚Äî should match the actual game result

**Why it's trustworthy:**
- Casino can't change outcome after you bet (server seed was hashed beforehand)
- You can verify every round yourself (independence from casino)
- If verification fails, the game is rigged

**What to look for:**
- ‚úÖ Server seed is hashed and disclosed *before* you bet
- ‚úÖ Client seed is generated by YOU (not the casino)
- ‚úÖ Verification tool is built-in (no external scripts needed)
- ‚úÖ You can run 1,000+ simulations to verify RTP

## Fake Provably Fair: How Casinos Cheat

### Scam #1: Fake Hash

**How it works:** Casino displays a hash, but it's not actually the hash of the server seed. It's a random string that looks legit.

**Detection:**
- After the round, verify that the revealed server seed actually produces the hash
- If `SHA256(server_seed) != displayed_hash`, the game is rigged

**Real example (2024 scam):**
A crash casino displayed server seed hashes, but when seeds were revealed, the hashes didn't match. The casino was faking provably fair entirely. **Players lost ~$300,000 before the site shut down.**

---

### Scam #2: Predictable Client Seed

**How it works:** Casino generates the client seed for you (instead of you generating it). Since they know both server seed and client seed in advance, they can predict the crash point.

**Detection:**
- Check if client seed is user-generated
- If casino sets it for you, it's rigged

**Why this matters:** If casino knows both seeds, they can calculate crash points before you bet and adjust to minimize payouts.

---

### Scam #3: Fake Verification Tool

**How it works:** Casino provides a verification tool that *always* returns "verified" regardless of input. It gives false confidence.

**Detection:**
- Run verification with obviously wrong data (server seed = "test")
- If tool still says "verified," it's fake

**Real example (2023 scam):**
A crash casino's verification tool was a JavaScript function that always returned `true` regardless of input. Players thought they were verifying, but they weren't. **Scam uncovered by a Reddit user who inspected the code.**

---

### Scam #4: Actual RTP Doesn't Match Advertised

**How it works:** Casino claims 97% RTP but actual RTP is 85-90%. Provably fair "works" but the game is mathematically rigged against you.

**Detection:**
- Play 1,000+ rounds
- Record every crash point
- Calculate actual RTP: (Total won / Total wagered) √ó 100
- If RTP is 85-90% (not 97%), the game is rigged

**Why this matters:** Even if verification passes, a lower RTP means you lose faster.
