## How Provably Fair Works

### The Concept

Provably fair means the casino commits to the outcome *before* you bet. They can't change it after you place your bet.

**Here's how it works:**

1. **Before you bet:** Casino generates a random server seed, hashes it, and publishes the hash
2. **You place your bet:** You provide (or choose) a client seed
3. **Crash point is calculated:** Server seed + client seed + nonce (bet number) = crash point
4. **After the round:** Casino reveals the unhashed server seed
5. **You verify:** You can recreate the crash point yourself using the seeds + nonce

**If your calculated result matches the casino's result, the game was fair.**

### Why It's Trustworthy

- **Casino can't change outcome:** Server seed hash was published *before* you bet
- **You can verify independently:** You don't have to trust the casino ‚Äî you can run the math yourself
- **Transparent process:** All seeds and nonces are disclosed

### What to Look For (Transparent vs Opaque Casinos)

**‚úÖ Transparent Casinos (Good Signs):**
- Server seed hash is disclosed BEFORE you bet (usually in "Fairness" or "Provably Fair" page)
- Client seed is generated by YOU (not the casino)
- Verification tool is built-in (no external scripts needed)
- You can run 1,000+ simulations to verify RTP
- Support can explain provably fair in detail

**‚ùå Opaque Casinos (Red Flags):**
- No hash disclosure before you bet (they can change outcome after)
- Client seed is generated by casino (they can rig it)
- No verification tool (you have to "just trust" them)
- Support can't explain how provably fair works
- Vague "we're fair" claims without proof

## Fake Provably Fair: How Casinos Cheat

Just because a casino *claims* provably fair doesn't mean it's true. Here's how casinos fake it:

### Scam #1: Hash is Fake

**How it works:** Casino shows a server seed hash, but it's not actually used in the game. The real RNG is hidden.

**Detection:**
- Verify multiple bets (5-10)
- If ANY fail verification, the hash is fake
- Calculate actual RTP over 100 bets (should be ~97%)

**Real example:** CrashKingz showed seed hashes, but independent audits showed the hashes weren't used in game logic.

### Scam #2: Client Seed is Predictable

**How it works:** Casino generates the client seed for you (not you yourself). If they know your seed in advance, they can calculate outcomes and rig crashes.

**Detection:**
- Check if client seed is generated by YOU (not casino)
- If you can't input your own seed, it's a red flag

**Real example:** Some shady casinos use timestamp-based client seeds (predictable ‚Äî they know when you'll bet).

### Scam #3: Verification Tool is Rigged

**How it works:** The verification tool *always* shows "verified" even when the game is rigged.

**Detection:**
- Run 1,000 simulations using the verification tool
- Calculate actual RTP from simulations
- If RTP is 85-90% (not 97%), the tool is rigged

**Real example:** LuckyCrash's verification tool showed "verified" for all bets, but independent analysis showed actual RTP was 83%.

### Scam #4: No Hash Disclosure Before Bet

**How it works:** Casino only reveals the server seed *after* you bet (not before). This allows them to change the seed based on your bet.

**Detection:**
- Check if server seed hash is published BEFORE you bet
- If hash only appears after bet, it's useless

**Why this matters:** The whole point of provably fair is that the casino commits to the outcome *before* you bet. If they reveal the seed after, they can change it.
